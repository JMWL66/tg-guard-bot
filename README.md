# 🛡️ tg-guard-bot (終極究極版 — Ultimate Master Edition)

這是一個基於 **Cloudflare Workers** 邊緣運算開發的工控級 Telegram 群組管理員。它具備「零容忍」過濾、高效快取與深度掃描功能，能有效保持群組環境純淨。

---

## 🔥 核心旗艦功能 (Core Features)

### 1. 🚫 永久封鎖處置 (Permanent Ban)
Bot 會針對以下違規行為執行 **「永久封鎖 (Ban)」**，用戶被封後無法自行重新加入（需管理員手動 `/unban`）：
*   **轉發訊息**：嚴禁任何頻道、外部群組或匿名用戶的轉發。
*   **非白名單連結**：偵測到非授權網址、`t.me` 連結或 `@提及` 模式立即封鎖。
*   **洗版/重複訊息**：自動偵測高頻率發言、重複長訊息或連續無意義短訊息。

### 2. 🛡️ 違規累計與保護 (Violation Tracking)
*   **累計計數**：系統會透過 KV 自動記錄每位成員的違規次數 (7 天滑動視窗)，並顯示於管理員日誌中。
*   **新成員保護**：新成員入群前 5 分鐘禁止發送任何連結。
*   **入群歡迎**：新成員加入時顯示簡潔的保護提示（具備 10 秒冷卻期，多人同時入群也僅發送單次提醒）。

### 3. 🔍 深度掃描引擎 (Deep Scan)
*   **Regex + Entities**：雙重偵測原始文字與 Telegram 實體連結（隱藏連結也無所遁形）。
*   **多方位監控**：同步過濾一般訊息、編輯過的訊息、媒體說明 (Caption) 以及頻道轉發。

### 4. 🚀 效能與隱私 (Performance & Privacy)
*   **管理員快取 (Admin Cache)**：自動快取管理權限 1 小時，將 API 延遲降至最低。
*   **增強型日誌**：違規記錄會同步發送私訊給系統管理員 (`ADMIN_ID`)，包含 **被刪除的原文預覽** 與 **違規累計次數**。
*   **隱形指令**：所有管理員指令執行後，指令本身與 Bot 的回覆皆會 **「自動刪除」**，保持群組版面純淨。

## 🛠️ 管理員指令指南 (Admin Commands)

僅限 **建立者** 或 **管理員** 使用。所有管理指令執行後將 **自動刪除指令本身**。

| 指令 | 功能說明 | 範例 |
| :--- | :--- | :--- |
| `/help` | 顯示所有可用管理指令與說明 | `/help` |
| `/allow <domain>` | 添加域名到動態白名單（例如放行搜尋引擎） | `/allow google.com` |
| `/disallow <domain>` | 將域名從動態白名單移除 | `/disallow google.com` |
| `/listwhitelist` | 查看系統規則 + 動態放行域名 | `/listwhitelist` |
| `/allowuser <uid>` | 將特定用戶 UID 加入特許白名單（完全豁免） | `/allowuser 123456` |
| `/disallowuser <uid>` | 將用戶從特許白名單中移除 | `/disallowuser 123456` |
| `/unban <uid>` | 手動解封被封禁的成員（UID 可從日誌複製） | `/unban 1234567` |
| `/resetviolations <uid>` | 清空特定成員的違規計數與黑紀錄 | `/resetviolations 1234567` |
| `/resetadmincache <uid>` | 手動重置指定 UID 的管理員快取 | `/resetadmincache` |

## 🛠️ 配置祕笈 (Configuration)

### 1. 設置 Secrets (Wrangler)
執行以下指令將敏感資訊上傳至 Cloudflare：
```bash
npx wrangler secret put BOT_TOKEN           # 你的 Telegram Bot API Token
npx wrangler secret put ADMIN_ID            # 你的個人 UID (接收違規日誌)
npx wrangler secret put WEBHOOK_SECRET_TOKEN # (選填) Webhook 安全密鑰
```

### 2. KV 命名空間
需手動建立一個 KV 命名空間並在 `wrangler.toml` 中綁定為 `TG_GUARD_KV`。

## 🧩 常見問題 (FAQ)

### Q1: 如何找到自己的數字 Admin ID？
請私訊 Telegram 的官方工具機器人 **`@userinfobot`**，發送任何訊息給它，它會回傳你的 `Id`（例如 `580979387`）。

### Q2: 為什麼我已經設定了 ADMIN_ID 但收不到違規日誌？
因為 Telegram 限制了 Bot 不能主動私訊。請先主動與你的 Bot 開啟對話（點擊 **Start**），之後有人違規時，Bot 才有權限發送私訊給你。

### Q3: 指令發出後沒反應怎麼辦？
1.  請確認你的 Bot 已被設為管理員，且擁有「刪除訊息」與「封禁成員」權限。
2.  在群組中輸入 `/help`，如果指令訊息自動消失且 Bot 發出 5 秒自動消失的回覆，表示正常運作。

## 🧪 測試清單 (QA Checklist)
- [ ] **域名放行測試**：測試 `/allow` 後非管理員是否能發送該網址。
- [ ] **編輯偵測測試**：普通文字訊息被編輯成連結後，是否立刻消失。
- [ ] **隱現指令測試**：發送指令後，確認指令消失、Bot 回覆隨後也消失。
- [ ] **累計次數測試**：測試違規多次後，管理員日誌顯示的計數是否增加。
- [ ] **封鎖與解封測試**：測試違規被封後，管理員執行 `/unban` 是否能解除限制。

## 🛠️ 最近修復與優化 (Recent Fixes & Improvements)

### 1. 處理「已刪除帳號 (Deleted Account)」的違規行為
*   **問題背景**：當已刪除帳號（`message.from` 為 `null`）發送轉發訊息或連結時，舊程式碼會因找不到 `userId` 而提早結束處理，導致違規訊息無法被自動刪除。
*   **修復方案**：移除 `handleMessage` 入口處對 `userId` 的強制性檢查。
    *   **轉發與連結規則**：現在允許 `userId` 為 `null` 的消息進入檢查流程。若偵測到違規，會立刻執行 **「刪除消息」**，但會自動跳過針對該用戶的 **「封禁 (Ban)」** 與 **「違規次數累計」** 操作（避免 API 報錯）。
    *   **洗版偵測規則**：由於洗版偵測依賴 `userId` 作為唯一鍵值，若 `userId` 為 `null` 則會安全跳過此項檢查。

### 3. 關鍵詞與轉發偵測優化 (Keyword & Forward Detection Enhancements)
*   **關鍵詞顆粒度優化**：將原本組合式的關鍵詞（如「內部群+社區」）拆分為單一詞觸發。現在「社區」、「內部群」、「跟單」等詞只需單獨出現即可觸發刪除並封禁。
*   **廣泛轉發檢查**：除了 `forward_origin`，額外同步檢查 `forward_date`, `forward_from`, `forward_from_chat` 等欄位，確保無論發言者如何隱藏，轉發訊息皆會被攔截。
*   **變體關鍵詞擴充**：新增對「返用」、「直播間」、「跟單老師」等常見廣告變體詞的偵測。

### 4. 安全漏洞修復與防禦 (Security Patches)

針對廣告商可能的「繞過方式」，我們已完成以下防禦部署：

| 繞過方式 | 觸發條件 | 影響 | **修復狀態** |
| :--- | :--- | :--- | :--- |
| **引用回覆替代轉發** | 使用回覆鍵而非轉發 | 規避規則 1 (轉發偵測) | **已修強** (加強回覆內容掃描) |
| **外部頻道匿名身分** | 以外部頻道身分發言 | 全部規則跳過 (偽裝管理員) | **已修復** (嚴格校驗匿名身份) |
| **引用無標題媒體** | 引用回覆無 Caption | 規避規則 2 (關鍵詞) | **已優化** (深度檢查媒體元數據) |

*   **身份校驗優化**：現在機器人會嚴格比對 `sender_chat`。只有當發言頻道與本群組 ID 一致（即真正的「匿名管理員」）時才會放行；所有外部頻道身分現在皆會經過關鍵詞與連結過濾。
*   **回覆深度掃描**：對回覆（Reply）訊息的文字、連結與媒體說明進行更深層的 Regex 與 Entities 偵測。

### 5. 系統帳號絕對豁免與權限邏輯精煉 (System Account Exemption & Logic Refined)
*   **絕對豁免部署**：在 `handleMessage` 最頂部加入「第一道防線」，針對 `GroupAnonymousBot` (ID `1087968824`) 等系統帳號執行絕對豁免，繞過後續所有偵測邏輯。保證系統功能（如以頻道身分發言）不被誤判。
*   **權限判定順序優化**：修復了 `getMemberStatus` 在處理 `sender_chat` 時可能過早返回 `member` 的邏輯漏洞。
*   **修正後的判定優先級**：
    1.  **匿名管理員** (sender_chat 是本群) → `administrator`
    2.  **系統帳號優先** (SYSTEM_BOT_IDS) → `administrator` (關鍵：移至外部頻道 fallback 之前)
    3.  **外部頻道** (非系統帳號) → `member`
    4.  **其餘用戶** → 查詢快取或 Telegram API

### 6. 精細化 t.me 連結識別邏輯 (Refined Telegram Link Classification)
已部署。為了平衡安全與社交需求，我們對 `t.me` 域名下的網址進行了分類識別：

| 連結類型 | 例子 | 處理方式 |
| :--- | :--- | :--- |
| **邀請類** | `t.me/+abc123` / `t.me/joinchat/xxx` | **封鎖** (嚴格防範私有廣告群) |
| **公開類** | `t.me/durov` / `t.me/somebot` / `t.me/channel` | **封鎖** (防止外部引流) |
| **白名單關鍵詞** | `t.me/paperworkbybryan` | **放行** (系統信任路徑) |

### 7. 深度防護性能擴充 (Extended Security & Detection)

針對近期垃圾信息變體，我們進行了以下核心升級：

*   **關鍵詞表擴充（根本原因分析）**：
    *   新增針對「實盤帶單」、「輕鬆跟車」、「佈局空單」、「多空信號」等誘導性文字的偵測。
    *   完整支援簡繁體對應版本，有效攔截詐騙引流訊息。
*   **新增 Emoji 特殊過濾**：
    *   現在任何訊息裡含有 **2 個或以上 ✅** 都會觸發關鍵詞規則，直接刪除訊息並執行永久封禁。
    *   (此規則有效打擊廣告商利用 ✅ 表格作為誘導引流的手段)。
*   **修復 Entities 提取漏洞**：
    *   解決了原本 `message.entities || message.caption_entities` 在 `entities` 為空陣列 `[]` 時會導致 `caption_entities` 被完全忽略的邏輯漏洞。
    *   現在能確保圖片、影片等媒體訊息的說明（Caption）連結被 100% 掃描。
*   **擴展關鍵詞掃描範圍**：
    *   監控範圍現在包含 **「轉發來源的頻道名稱 (Forward Origin Chat Title)」**。
    *   例如：轉發自名為「比特幣 btc 合約免費跟單」頻道的任何訊息，將會因觸發頻道名稱關鍵詞而被自動攔截。
### 8. 特許用戶豁免機制 (User Whitelisting & Exemption)

*   **為了提供更靈運的管理，我們新增了 **「用戶層級」** 的白名單機制：

*   **完全豁免權**：被加入特許名單的用戶將會繞過 **所有限制**（包括轉發、連結、關鍵詞、洗版與短訊息偵測）。
*   **硬編碼 + 動態組合**：支援在程式碼中預設特許用戶，也支援透過管理員指令 `/allowuser` 隨時動動態增減。
*   **透明化管理**：特許用戶名單會同步顯示於 `/listwhitelist` 列表中，方便管理員隨時視察。



### 9. 階梯式處罰系統 (Tiered Punishment System)
為了提供更人性化的管理，我們將原本的「直接剔除」改為三階段階梯式處罰：
| 違規次數 | 處罰措施 | 詳細限制 | 恢復時間 |
| :--- | :--- | :--- | :--- |
| **第 1 次** | **禁言 24 小時** | 允許發送純文字，但禁止媒體 (圖片/影片)、連結與轉發 | 24 小時後自動恢復 |
| **第 2 次** | **禁言 7 天** | **完全禁止**在本群發送任何訊息 | 7 天後自動恢復 |
| **第 3 次** | **永久封鎖** | 直接移出群組並列入黑名單 | 需聯絡管理員手動解封 |

### 10. 全量活動報表 (Cumulative Activity Report)
系統現在會自動累計所有歷史違規紀錄，並多管道提供 Excel 相容報表：
*   **網頁即時下載**：透過私密連結（含 Secret Token）隨時在瀏覽器下載最新全量報表。
*   **每日自動備份**：系統每天 UTC 00:00 會自動將最新全量報表發送至管理員私訊。
*   **指令手動匯出**：管理員可隨時透過 `/exportall` 獲取最新檔案。

### 11. 審核機制優化與精準過濾 (Audit Optimization & Precise Filtering)
針對近期廣告商的規避手段，我們進行了深層邏輯重構，提升了系統的精準度與友善度：
*   **鏈接黑名單模式**：從原本的「全封鎖」優化為「特定黑名單 + 通用邀請鏈接封鎖」。現在一般正常連結（如外部網站或普通 `@username`）可以正常發送，精準打擊廣告商的特定推廣連結與私有群組。
*   **關鍵詞去敏化**：將容易誤報的單一關鍵詞（如「社區」、「跟單」、「帶單」）優化為組合觸發（例如需與「免費」、「點位」等詞同時出現才封鎖），確保正常技術討論不受影響。
*   **應對空格與符號規避**：新增訊息正規化處理，自動移除空格、標點與特殊符號（如 `币 圈 免 费` 或 `币.圈.免.费`）後再進行比對，讓任何形式的空間規避與字符偽裝失效。
*   **廣告語料庫擴充**：根據近期垃圾廣告樣本，新增了針對「合約交流裙」、「策略分享」、「不求一夜暴富」、「8年合約頂級社區」等精準誘導話術的過濾。
